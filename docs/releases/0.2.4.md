# mutsu 0.2.4 — Firefox (AMO) release & reviewer notes

> This is the **rebrand** release (previously published as **sixseven**). Same
> add-on, renamed. The add-on id is intentionally unchanged so existing installs
> update in place. (0.2.5, staged next, adds an in-page "update available" nudge.)

## Release notes (user-facing — AMO "What's new")

- **Renamed to mutsu** (formerly sixseven) — same extension, new name.
- Watching on sites that block embedding (Netflix-style) is much smoother: a
  clear **"Watch in your own tab"** path, your already-open tab is **paired
  automatically**, and you can **"Play this page for everyone"** to move the whole
  room to a new title.
- In-tab widget: cleaner header (room-name link to jump to the room, **minimize**
  to a bubble, **close**); the toolbar popup shows its controls instantly.
- Fixes: "Reopen tab" no longer opens duplicate tabs; the widget is removed when
  the room switches to an embeddable source.

## Notes for reviewers

### Rename / identity (please note)
This add-on was previously listed as **"sixseven"**. It has been **renamed to
"mutsu"** (a branding change only). The **add-on id is unchanged**
(`sixseven@onesal.me`) on purpose, so existing users update in place rather than
getting a separate listing — that's why the manifest `name` ("mutsu") differs from
the gecko id ("sixseven@onesal.me"). The companion web app moved to
`https://mutsu.onesal.me`.

### What the extension does
mutsu keeps a group's **video playback in sync** for a watch party. Everyone plays
their own copy of the same source in their own browser; the extension reads and
nudges the `<video>` element's **clock** (play / pause / seek / current time) so
playback stays aligned. It does **not** download, record, relay, re-encode, or
unlock any video — no media bytes pass through any server. It only moves a URL and
a timestamp. A companion web page (the "room", `https://mutsu.onesal.me`) holds the
group connection.

### Data collection & privacy
- **No data collection. No analytics, telemetry, tracking, ads, or cookies.**
- Only the chosen source **URL** and the **playback position** are sent to the room
  server so peers can sync — no personal data, browsing history, page content, or
  video.
- Declared `data_collection_permissions: { required: ["none"] }`.
- No accounts; a nickname is stored in `localStorage` on the web page only.

### No remote code
All logic ships in the package — **no remote scripts**, no `eval`/remote code
execution. It connects only to the room server for sync and (on user request)
subtitles/GIFs via that server.

### Permission justifications
- **`host_permissions: <all_urls>` + content script on all frames** — the source a
  group watches can be *any* site/embed provider chosen at runtime, so the content
  script must run in any page/iframe to find and sync its `<video>`. It stays inert
  on pages that aren't part of an active party.
- **`scripting`** — the toolbar popup scans the *active* tab for `<video>` /
  `<iframe>` sources so the user can pick what to watch together.
- **`tabs`** — for "watch this page in its own tab" parties, a background service
  worker relays sync messages between the user's room tab and their streaming tab
  (the only context that can message between two tabs) and notices when either
  closes. Also used to find the open room tab to deliver a picked source.
- **`storage`** — local tab-pairing state (which room tab drives which streaming
  tab). Session-scoped; cleared on browser close.
- **`web_accessible_resources: icon/*.png`** — the in-tab widget's floating bubble
  shows the extension icon, which a content-script `<img>` can only load if the
  icon is web-accessible.

### How to test
1. Open `https://mutsu.onesal.me` (open it in two browser profiles to simulate two
   people).
2. Click the mutsu toolbar button → "New room", or paste a video/embed URL in the
   room's **Source** panel.
3. Play/pause/seek in one window and observe the other staying in sync.
4. "Watch in your own tab": pick a site that blocks embedding; the room shows
   "Watch in your own tab", which opens/pairs a tab the extension then syncs.

No login or paid account is required.

### Reproducing the build (AMO source review)
- Environment: Node.js ≥ 22, pnpm 9.15.0 (see `packageManager` in the root
  `package.json`), any OS (built on macOS).
- From the repository root:
  ```bash
  pnpm install
  pnpm --filter @mutsu/extension build:firefox   # → .output/firefox-mv2
  # (or `… zip -b firefox` to reproduce the submitted zips)
  ```
- Build output is minified by the bundler; the **sources zip** (`*-sources.zip`)
  is included. Source repo: `https://github.com/kasusoba/mutsu`. No build-time
  secrets are required — the room host is a public URL baked into the build
  (`mutsu.onesal.me`), not a credential.
